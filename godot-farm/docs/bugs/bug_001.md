# Bug Report: SaveManager Missing

## ID
BUG-001

## Severity
🔴 Critical - Crash/data loss

## Environment
- Godot Version: 4.6
- Platform: All
- Commit: current

## Description
The SaveManager autoload script is referenced in `project.godot` but the file does not exist. This will cause the game to crash on startup.

## Reproduction Steps
1. Open project in Godot
2. Run the game
3. Observe crash: "Cannot autoload 'res://scripts/systems/save_manager.gd'"

## Expected Behavior
Game should start with SaveManager loaded as singleton.

## Actual Behavior
Game crashes on startup due to missing script file.

## File Reference
```
[autoload]
...
SaveManager="*res://scripts/systems/save_manager.gd"
```

## Required Implementation
- Save/load game state to disk
- Manage save file slots
- Handle save migration/versions
- Integration with StateManager.serialize()/deserialize()
