# Binance 全自动高频网格交易系统 — 策略文档

## 系统概览

全自动杠杆网格交易系统，目标：7x24 无人值守，在 Binance 合约市场通过高频微网格捕捉震荡利润，同时用多层风控系统防御单边行情和黑天鹅事件。

---

## 一、核心交易策略：自适应微网格

### 1.1 网格间距算法

```
实际间距 = max(0.05%, Spread × 1.1, BollingerBandwidth / 12)
```

| 条件 | 间距行为 | 目的 |
|------|---------|------|
| 死寂行情（低波动） | 收窄至 ~0.05% | 覆盖手续费后微利，高频刷量赚积分 |
| 正常波动 | 跟随 BB 带宽 | 匹配市场节奏 |
| 剧烈波动 | 自动张开 | 减少被打穿的概率 |

### 1.2 ADX 动态呼吸护盾（V1.1 新增）

```
最终间距 = 实际间距 × (1 + ADX强度 × 乘数)
```

- 指标：14 周期 ADX（平均趋向指标）
- 效果：当市场从震荡转为强单边时，网格自动"吹气"撑大间距
- 价值：延缓加仓速度，在单边行情中接货位置更深更安全

### 1.3 结构化中轴锚定（V1.1 新增）

**问题**：中轴过于灵敏 → 插针/暴涨时频繁移动中心位 → 卖飞

**解决方案**：

| 时段 | 中轴锚定周期 | 理由 |
|------|------------|------|
| 非交易时段（亚盘等） | 5min BB 中轨 | 波动小，短周期足够 |
| 主要交易时段（美盘） | 15min BB 中轨 | 抵抗美股开盘深V的冲击 |

**效果**：反转回到均值时能吃到完整利润，不会因中轴漂移而提前卖出。

## 二、风控体系（机构级）

### 2.1 动态平衡模式

| 市场状态 | 杠杆策略 | 网格策略 |
|---------|---------|---------|
| 低波动期 | 高杠杆（标准模式） | 正常间距，高频刷量 |
| 不确定期 | 降杠杆 | 主动拉大网格间距 |
| 盘中高波动 | 最低杠杆 | 最大间距，保守运行 |

### 2.2 智能休眠

- 接入宏观经济日历（CPI、FOMC、非农等）
- 重大数据发布前 N 分钟：自动清仓/减仓
- 发布后观察期结束：自动恢复运行

### 2.3 AI 舆情熔断

```
流程：
1. Tavily 搜索 → 关键词命中检测
2. Gemini 语义分析 → 判断"即时风险"等级
3. 若确认风险 → 立即 STOP + 冷却 1 小时
```

- 引擎：Tavily Search + Google Gemini
- 触发条件：检测到敏感词 **且** AI 判定为即时风险（双重确认，避免误熔断）
- 冷却：1 小时后自动恢复

### 2.4 硬性止损

- 最大持仓上限（防无限加仓）
- 日亏损上限（超过阈值当日停止交易）
- 总账户回撤上限（超过阈值全面停止）

## 三、运维架构

### 3.1 双重守护

```
launcher.py  → 死循环拉起主脚本（进程级守护）
watchdog.sh  → Cron 定时检查 launcher 是否存活（系统级守护）
```

**确保 7x24 在线**：即使主脚本崩溃、launcher 崩溃，watchdog 都能在下一次 cron 执行时拉起。

### 3.2 远程控制

- 移动端提醒：异常/熔断时推送通知
- 移动端控制：支持远程修改策略参数、手动停止/启动
- OpenClaw 集成：通过 WhatsApp 接收状态报告和发送指令
